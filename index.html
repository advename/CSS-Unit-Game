CSS UNITS GAME

Learn progressivly the length units of the web

SCROLL

CSS or the style sheet of a website has many length units, same as we have centimeters (cm), millimeters (mm), inches (in) and so on and so forth in the real world. While we can use the units cm, mm and in with our style sheet, they are yet not recommended as they are not friendly to use with Web sites. Moreover, as Web sites are displayed on different screen sizes, we want to work relatively to the screen size of the device displaying the website.



Therefore, in CSS we work with two types of length units:

  • relative (depends on screen- / element-size)
  • absolute (fixed size)

There are over 10 length units compatible with CSS, however, most of the times we're only going to need a couple of them:

Unit Description Example
px Pixels - absolute unit 1px = 1/96th of 1in
em Relative to the font size of the element 2em means 2 times the size of the elements font
vh Viewport Height - relative to the browser window height 50vh would fill half of the screen vertically
vw Viewport Width - relative to the browser window width 50vw would fill half of the screen horizontally
% Percentage - relative to the size of container enclosing the element Self explaining
auto Auto is the default length and depends on the elements inside the container. An element inside a container has 50px height and width. The container is set to auto ,height and width will also be 50px.

Tablet Example with unit lengths


Imagine this was a Tablet, which has a browser window with a height of 300px and a width of 260px. The font-size is 14px.


195px ( filling 75% of 260px )

12em ( equals to 168px )

84vw ( 84/100 of the browser width )

81% (81% of the containers width)

This container has a width of 81vw.
This box has a width of 50%

90vh (90/100 of browser height)

Depending on the screen size, only the 200px will keep same size. CSS length units are written with the length and then the unit without a space inbetween. vw and vh can both be used for height and width lengths.

What to use them for?


Besides defining the height and width of a container, we can also add a border to our container or elements, define the size of the fonts, and add space in two ways between the element and the container enclosing it:


  • margin, adding empty space around the element
  • padding, adding space from the inside of the element


Margin and padding are set with one, two or four values:


  • margin : 10px (is equal to a 10px empty space on every side around the element)
  • margin : 10px 20px (is equal to a 10px of empty space to the top and bottom, and 20px of empty space to the left and right of the element)
  • margin : 10px 20px 30px 40px (is equal to add 10px to the top, 20px to the right, 30px to the bottom and 40px of margin to the left)

The same rule applies for padding.


About CSS Unit Game

Working with websites, if it's a designer, programmer or copywriter, they all need to work with lengths to create stunning user interfaces and articles. However, not all of them have access to get easily understand how these length units work without a programming interface or analysing the codes behind. That's where this "game" comes in. By several objectives, I want to bring everyone working on the web closer to the understanding of digital units.
